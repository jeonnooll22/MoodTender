# MuseTalk 속도 최적화 기록

## 목표
텍스트 입력 후 영상 재생 시작까지 5초 이내

---

## 기준 성능 (최적화 전)

- 방식: 전체 영상 생성 완료 → 재생
- 영상 재생 시작까지: **~22초**
- GPU 사용률: ~75% (GPU 추론과 CPU 블렌딩이 순차 실행)

---

## 시도 1: batch_size 조정

GPU가 한 번에 처리하는 프레임 수를 늘려 배치 횟수를 줄이는 방식

| batch_size | 추론 시간 | 결과 |
|-----------|---------|------|
| 8 (기본)  | 21.9초  | ✅ 최적 |
| 12        | 23.7초  | 느려짐 |
| 16        | timeout | VRAM 부족 |

**결론:** 배치를 늘릴수록 배치당 시간이 늘어나 오히려 느려짐. batch_size=8 유지.

---

## 시도 2: GPU/CPU 병렬 스레딩

**문제:** GPU 추론 → CPU 블렌딩 순서로 실행되어 GPU가 CPU 작업 중 대기

```
기존:  [GPU 추론 0.7초] → [CPU 블렌딩 0.2초] = 0.9초/배치
변경:  [GPU 추론 0.7초]
           └── 동시에 ── [CPU 블렌딩 0.2초]  = 0.7초/배치
```

`stream_inference.py`에서 GPU 추론 스레드와 CPU 블렌딩 스레드를 분리,
Queue로 프레임을 주고받아 병렬 처리.

- GPU 피크 사용률: 75% → 98%
- **효과: 27.7초 → 23.7초 (14% 단축)**

---

## 시도 3: torch.compile

PyTorch의 JIT 컴파일로 UNet 추론 속도를 높이는 방식

```python
unet.model = torch.compile(unet.model, mode="reduce-overhead")
```

**결과: Windows 미지원으로 스킵**

- `inductor` 백엔드: Windows 미지원
- `aot_eager` 백엔드: 동일하게 실패
- Docker + WSL2(Linux) 환경에서는 사용 가능 → 추가 15~20% 개선 예상

---

## 시도 4: FFmpeg 실시간 파이프 스트리밍

전체 생성 완료를 기다리지 않고, 프레임이 생성되는 즉시 FFmpeg로 파이프해
fMP4 청크를 HTTP 스트림으로 전송. 브라우저는 MSE로 청크 단위 재생.

### FFmpeg 조각화 방식 실험

| 방식 | 첫 청크 | 재생 품질 |
|------|--------|---------|
| `frag_keyframe` `-g 25` (1초 간격) | 15.2초 | 양호 |
| `frag_every_frame` + `-fflags +nobuffer` | 8.7초 | 끊김 |
| `frag_every_frame` + `-tune zerolatency` | 1.8초 | 프레임 단위 끊김 |
| **`frag_keyframe` + `-g 12` (0.48초 간격)** | **~2초** | **양호** |

`frag_every_frame`은 초당 25개 조각을 생성해 MSE가 처리하기에 너무 세밀해 끊겨 보임.
0.48초 간격(`-g 12`) 조각이 부드러운 재생과 빠른 시작의 균형점.

**효과: 영상 재생 시작 22초 → 1.8초**

## 현재 적용된 최적화 조합

| 최적화 | 적용 여부 | 효과 |
|--------|---------|------|
| batch_size=8 | ✅ | 기본 최적값 |
| GPU/CPU 병렬 스레딩 | ✅ | ~14% 속도 향상 |
| FFmpeg frag_keyframe -g 12 | ✅ | 첫 청크 ~2초 |
| MSE 버퍼 모니터링 (자동 일시정지/재개) | ✅ | 끊김 UX 최소화 |
| torch.compile | ❌ Windows 미지원 | (Linux에서 +15~20%) |

---

## 현재 성능 (RTX 4060)

```
버튼 클릭
  ↓ 0.5초  TTS 변환
  ↓ ~1.3초 첫 3배치 GPU 추론 + FFmpeg 버퍼
  ↓        ──────────────────────────────
  → 1.8초  영상 재생 시작  ✅ (목표 5초 이내 달성)
  → 23.7초 전체 생성 완료 (7초 음성 기준)
```

버퍼링 발생 이유: 생성 속도(0.3x 실시간) < 재생 속도(1x 실시간)
→ 자동 일시정지/재개로 UX 처리
